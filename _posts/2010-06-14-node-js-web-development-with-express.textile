Yes, node has an ecosystem, a very vibrant one if I may say. Today I tried a simple web framework that resembles Ruby's Sinatra by the name of Express.
<h3 style="margin-bottom: 0cm;">Installation<br /></h3>  <p style="margin-bottom: 0cm;">There are 2 ways to install it, one using Kiwi a node package manager, the other is cloning the github repo and doing make install on it. I first tried using Kiwi but it didn't work for me so I got the code with git and didn't have a problem.
  <p style="margin-bottom: 0cm;">There is some documentation on the website, enough to get you started.
 <h3 class="western">Characteristics</h3> Most of the time you will write your application in one file that acts as a front controller. Actually is not a controller just a module,lets call it main. In the module you write methods to handle requests according to the HTTP VERB used. You map them to url's passing the pattern as the first parameter. For example.
 <p style="margin-bottom: 0cm; background: none repeat scroll 0% 0% rgb(0, 0, 0);" align="LEFT"><font color="#f8f8f2"><font face="Inconsolata"><font color="#f8f8f2">get(</font><font color="#ffff66">'/'</font><font color="#f8f8f2">,</font><font color="#8ae5ef"><i>function</i></font><font color="#fd971f">(){<br /> this</font><font color="#f8f8f2">.redirect(</font><font color="#ffff66">'/home'</font><font size="4">);<br />})</font></font></font>
   <p style="margin-bottom: 0cm;">Will handle a request to the root of your application. In the same module (file) you can have all the handlers for your site (if you want) even when they represent very different resources. I prefer to separate them in some more logic organization for most but the simple apps.<br />For example I want to have the code that handles all calls to the route /admin/* in an admin module, so I create a new file called admin.js and place this file in the same folder as my app.js file
  <p style="margin-bottom: 0cm;">Them at the top I just type.
  <p style="margin-bottom: 0cm; background: none repeat scroll 0% 0% rgb(192, 192, 192);"><font face="Courier 10 Pitch">require.paths.unshift('.')<br />require('admin')</font>
  <p style="margin-bottom: 0cm;">The unshifting of the paths is needed because we previously change it to the location of the express module you can also just type
  <p style="margin-bottom: 0cm; background: none repeat scroll 0% 0% rgb(192, 192, 192);"><font face="Courier 10 Pitch">require('./admin')</font>
 <h3 class="western">Running the application and debugging.</h3> Once you have all your handlers done starting the app is as easy as type:
 <p style="background: none repeat scroll 0% 0% rgb(192, 192, 192);"><font face="Courier 10 Pitch">node app.js</font>
 If you have any mistakes in your code you will see an error in the command line:
 <img style="max-width: 800px;" src="images/posts/console_error.png" /><br /><br />
 <p style="margin-bottom: 0cm;">If the error is found during run time, by default, Express provides a friendly 500 error page.
<p style="margin-bottom: 0cm;"><img style="max-width: 800px;" src="images/posts/500_error.png" /><br />
 <p style="margin-bottom: 0cm;">In this case the error is a view file missing.
  <h3 class="western">Views</h3> Out of the box Express knows about Haml and Saas templates and will look for view files in the views folder. It also support layouts and partial views to compose complex ui.
 By default will try to get a layout by the name of "layout" for the same view engine you are using for the view, for example in our case that we are using haml, it will try to load a file by the name of layout.html.haml. <br />You can disable layouts, setting layout: false in the hash passed to the render method of the request object.
 <p style="margin-bottom: 0cm; background: none repeat scroll 0% 0% rgb(0, 0, 0);" align="LEFT"><font face="Inconsolata"><font><font color="#f8f8f2">get(</font><font color="#ffff66">'/home'</font><font color="#f8f8f2">, </font><font color="#8ae5ef"><i>function</i></font><font color="#75715e">(){<br />//return 'Hello you are at the home of the new Express site'; <br /></font></font></font><font color="#f8f8f2"> </font><font color="#f8f8f2"><font face="Inconsolata"><font color="#fd971f">this</font><font color="#f8f8f2">.render(</font><font color="#ffff66">'home.html.haml'</font><font>,{<br />layout: </font><font color="#ae81ff">false</font><font>,<br /> locals: {<br /> title: </font><font color="#ffff66">'Express demo site'</font><font>,<br /> message: </font><font>'This welcome message is brought to you thank to Haml'<br /> }<br />})<br />})</font></font></font>
  <font size="3">Or you can specify the name explicitly:</font>
 <p style="background: none repeat scroll 0% 0% rgb(0, 0, 0);"><font color="#f8f8f2">    <font><font face="Inconsolata">layout: </font></font><font color="#ffff66"><font face="Inconsolata">'general.html.haml'</font></font><font color="#f8f8f2"><font face="Inconsolata">,</font></font></font>
 <h3 class="western">Plug-ins</h3>  Express has a very simple plug-in architecture that make it very easy to extend. Express uses plug-ins to provide it's core services, so besides the 4 required core plug-ins you can add or remove services that you don't use.
