  Besides programming against Interfaces and using <a href="http://en.wikipedia.org/wiki/Dependency_injection" target="_blank">Dependency Injection</a> to reduce coupling, there is another practice that can/should be used when you consume an external library. That is provide a wrapper around the library or in other words the use of the <a href="http://en.wikipedia.org/wiki/Facade_pattern" target="_blank">Facade pattern</a>.
  <h3>Why?</h3>  Consider this example.
  <pre class="c#:collapse" name="code">using System;<br />using System.Collections.Generic;<br />using System.Text;<br />using log4net.Core;<br /><br />namespace HtmlHelpers<br />{<br />    public class UnorderedList<br />    {<br />        private readonly ILogger _logger;<br /><br />        public UnorderedList(ILogger logger)<br />        {<br />            _logger = logger;<br />        }<br /><br />        public string Render(IEnumerable&amp;lt;string&amp;gt; collectionToPrint)<br />        {<br />            try<br />            {<br />                _logger.Log(collectionToPrint.GetType(),Level.Info,&amp;quot;Starting to render the list&amp;quot;,null);<br /><br />                var list = new StringBuilder();<br />                list.Append(&amp;quot;&amp;lt;ul&amp;gt;&amp;quot;);<br /><br />                foreach (var item in collectionToPrint)<br />                {<br />                    list.AppendFormat(&amp;quot;&amp;lt;li&amp;gt;{0}&amp;lt;/li&amp;gt;&amp;quot;, item);<br />                }<br /><br />                list.Append(&amp;quot;&amp;lt;/ul&amp;gt;&amp;quot;);<br /><br />                _logger.Log(collectionToPrint.GetType(), Level.Info, &amp;quot;Finishing to render the list&amp;quot;, null);<br /><br />                return list.ToString();<br />            }<br />            catch (Exception e)<br />            {<br />                _logger.Log(collectionToPrint.GetType(),Level.Fatal,&amp;quot;Can't render the list&amp;quot;,e);<br />                throw;<br />            }<br />        }<br />    }<br />}</pre>

As you can see using the ILogger interface in the constructor give us the ability to pass any object that implements it, but take a look at the using statements.


<pre class="c#:collapse" name="code">using System;<br />using System.Collections.Generic;<br />using System.Text;<br />using <font color="#ff0000">log4net.Core</font>;</pre>

Yeah, Log4Net is there, so even when we are isolating ourselves from a dependency to the concrete implementation, we are still carrying a dependency to the assembly. Not only that but if we want to change our logging service we need to do it for some other logging service that implements the same interface. 


As well you are having a dependency on the Level enumeration used in the signature of the Log method from the ILogger interface.


<pre class="c#:collapse" name="code">_logger.Log(collectionToPrint.GetType(),<font color="#ff0000">Level.Fatal</font>,&amp;quot;Can't render the list&amp;quot;,e);</pre>

While we are at this, take a look at that signature, you may want to do something simpler to use. So what will be the chance for another third party Logging library to implement this ILogger interface? Null of course. 


The solution is the Facade pattern. We start defining our own interface for logging.


<pre class="c#:collapse" name="code">using System;<br /><br />namespace FacadePattern<br />{<br />    public interface ILogger<br />    {<br />        void LogException(Exception e);<br />        void LogException(string message, Exception e);<br />        void LogInfo(string message);<br />    }<br />}</pre>

Notice how we create specific method for the levels we want to use, you may need other methods later on, but you can always implement them when you actually need them.
  <br />Now, we create a concrete implementation that uses Log4Net.


<pre class="c#:collapse" name="code">using System;<br />using System.Diagnostics;<br />using System.IO;<br />using System.Reflection;<br />using log4net;<br />using log4net.Config;<br /><br />namespace FacadePattern<br />{<br />    public class Logger : ILogger<br />    {<br />        private readonly ILog _logger;<br /><br />        public Logger()<br />        {<br />            XmlConfigurator.ConfigureAndWatch(new FileInfo(Path.Combine(<br />                                                               AppDomain.CurrentDomain.SetupInformation.ApplicationBase,<br />                                                               &amp;quot;log4net.config&amp;quot;)));<br />            MethodBase m = new StackTrace().GetFrame(1).GetMethod();<br />            _logger = LogManager.GetLogger(string.Format(&amp;quot;{0}.{1}&amp;quot;, m.DeclaringType, m.Name));<br />        }<br /><br />        public void LogException(Exception e)<br />        {<br />            _logger.Error(e.Message, e);<br />        }<br /> <br />        public void LogException(string message, Exception e)<br />        {<br />            _logger.Error(message, e);<br />        }<br /><br />        public void LogInfo(string message)<br />        {<br />            _logger.Info(message);<br />        }<br />    }<br />}</pre>

Now our objects that need Logging can consume our own implementation and any dependency on Log4Net is removed. At this point we can use Dependency Injection to provide our concrete implementation of the logging class and we are free to change the library that does the actual logging anytime we want.


<pre class="c#:collapse" name="code">using System;<br />using System.Collections.Generic;<br />using System.Text;<br />using FacadePattern;<br /><br />namespace HtmlHelpers<br />{<br />    public class UnorderedList<br />    {<br />        private readonly ILogger _logger;<br />        public UnorderedList(ILogger logger)<br />        {<br />            _logger = logger;<br />        }<br /><br />        public string Render(IEnumerable&amp;lt;string&amp;gt; collectionToPrint)<br />        {<br />            try<br />            {<br />                _logger.LogInfo(&amp;quot;Starting to render the list&amp;quot;);<br /><br />                var list = new StringBuilder();<br />                list.Append(&amp;quot;&amp;lt;ul&amp;gt;&amp;quot;);<br /><br />                foreach (var item in collectionToPrint)<br />                {<br />                    list.AppendFormat(&amp;quot;&amp;lt;li&amp;gt;{0}&amp;lt;/li&amp;gt;&amp;quot;, item);<br />                }<br /><br />                list.Append(&amp;quot;&amp;lt;/ul&amp;gt;&amp;quot;);<br /><br />                _logger.LogInfo(&amp;quot;Finishing to render the list&amp;quot;);<br /><br />                return list.ToString();<br />            }<br />            catch (Exception e)<br />            {<br />                _logger.LogException(&amp;quot;Can't render the list&amp;quot;,e);<br />                throw;<br />            }<br />        }<br />    }<br />}</pre>

<pre class="c#:collapse" name="code"><br />Also notice how much more clean and readable is the log code.</pre>
