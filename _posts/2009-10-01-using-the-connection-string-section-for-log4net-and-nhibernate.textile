The other day at the office my boss took a look at our configuration files and he expressed his concern about breaking DRY with all the ConnectionStrings. One for Ado.Net, one for NHibernate, one for our caching db and another for Log4Net. Not only that but they are all over the place in the file, not all together.
 On top of that,, some of those are the same, so changing a username or a password means changing them all over. Of course he was right.
 <h3>NHibernate solution</h3> For NHibernate I just use the connection.connection_string_name property instead of the connection.connection_string. This points to a connection string defined in that section of the configuration file with the provided name.

<h3>Log4Net solution</h3>
For Log4Net was not so easy. I found references to a patch that provides similar functionality to the one observed in NHibernate and references to that patch been applied to the latest release, but It doesn’t seem to be there.

So the solution I came with is to extend the default AdoNetAppender and in the constructor of the new class try to read the connection string from the configuration file, the only problem with this approach is that I’m hard coding the expected name of an AppSetting that will contain the name of the connection string.

Note: Utils.GetAppSetting and Utils.GetConnectionString are utility methods that I use to read from the configuration files.
<pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px"><span style="color: #0000ff">using</span> log4net.Appender;
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px"></pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px"><span style="color: #0000ff">namespace</span> ACME.Logging
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">{
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">	<span style="color: #0000ff">public</span> <span style="color: #0000ff">class</span> CustomAdoNetAppender : AdoNetAppender
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">	{
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">		<span style="color: #0000ff">public</span> CustomAdoNetAppender()
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">		{
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">			var stringKey = Utils.GetAppSetting("<span style="color: #8b0000">Log4NetConnectionStringName</span>");
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">			<span style="color: #0000ff">if</span> (<span style="color: #0000ff">string</span>.IsNullOrEmpty(stringKey)) <span style="color: #0000ff">return</span>;
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">			var connectionString = Utils.GetConnectionString(stringKey);
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">			<span style="color: #0000ff">if</span> (!<span style="color: #0000ff">string</span>.IsNullOrEmpty(connectionString)) ConnectionString = connectionString;
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">		}
</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">	}</pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px">}</pre></pre>
In your configuration file you need to have and appSetting that look like this:
<pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px"><span style="color: #0000ff">&amp;lt;</span><span style="color: #800000">add</span> <span style="color: #ff0000">key</span>=<span style="color: #0000ff">"Log4NetConnectionStringName"</span> <span style="color: #ff0000">value</span>=<span style="color: #0000ff">"ConnectionString"</span><span style="color: #0000ff">/&amp;gt;</span></pre></pre>
Them you will need to tell Log4Net to use this appender for logging so in your log4net.config file you should do something like this:
<pre><pre style="background-color: #ffffff; margin: 0em; width: 100%; font-family: consolas,'Courier New',courier,monospace; font-size: 12px"><span style="color: #0000ff">&amp;lt;</span><span style="color: #800000">appender</span> <span style="color: #ff0000">name</span>=<span style="color: #0000ff">"AdoNetAppender"</span> <span style="color: #ff0000">type</span>=<span style="color: #0000ff">"ACME.Logging.CustomAdoAppender"</span><span style="color: #0000ff">&amp;gt;</span></pre></pre>
